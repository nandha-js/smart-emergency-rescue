# Smart Emergency Rescue System (SERS)

An advanced, real-time emergency response platform utilizing a decoupled **MERN Stack** (MongoDB, Express, React, Node.js) integrated with **Edge AI** and **Google Gemini** for rapid distress verification, telemetry tracking, and responder routing.

---

## 🌟 Key Capabilities

### 1. Tactile SOS & Hands-Free Voice Trigger
*   **Tactile SOS Button**: Concentric pulsating panic button requiring a deliberate **3-second hold** to prevent false triggers, providing tactile device vibration feedback.
*   **Hands-Free Voice SOS**: Utilizes the **HTML5 Web Speech API** to continuously monitor microphone inputs for specific distress phrases (*"help"*, *"emergency"*, *"save me"*, etc.) and auto-trigger signals.

### 2. Acoustic Edge AI Trigger
*   **Audio Frequency Classifier**: Analyzes real-time microphone decibels using the Web Audio API to detect sudden acoustic distress signatures, including **glass breaking** (>4kHz), **screaming** (1kHz - 3.5kHz), and **collision crashes** (bass spikes >200 dB).

### 3. Interactive 3D "Virtual Paramedic" Avatar (Aria)
*   **Gemini-Powered Triage Chat**: Renders a dynamic 3D responder avatar on the victim's screen upon triggering SOS.
*   **Speech Synthesis & Lip-Sync**: Aria speaks calming, context-aware triage questions ("Are you safe?", "Where is the pain?") generated by Gemini and updates text logs dynamically.

### 4. Stealth Mode ("Silent Blackbox")
*   **Blackout Screen Overlay**: Allows victims to activate stealth mode, turning the screen completely black to appear powered down. It can be exited at any time by double-tapping the viewport.
*   **WebRTC Audio evidence streaming**: Silently records background microphone audio using the native `MediaRecorder` API and streams 5-second base64 segments to the dispatcher's feed.

### 5. Geofenced Danger Zones
*   **Dispatcher Drawing tool**: Mission control can click **📐 Draw Geofence**, tap any coordinate on the Leaflet map, input a name, and specify a radius (in meters) to save a Danger Zone GeoJSON to MongoDB.
*   **Boundary breach warning**: Active clients calculate distances in the background using the Haversine formula, displaying a pulsing warning banner if their GPS coordinates cross into a Danger Zone.

### 6. Twilio SMS & Smart Proximity Bystander Alerts
*   **SMS logs**: Automatically logs mock Twilio alerts containing live Google Maps links to emergency contacts.
*   **Smart Proximity Broadcaster**: If the Gemini-analyzed distress intent is a `medical_emergency`, the server runs a MongoDB `$near` query to find active users within 100 meters. Bystanders instantly receive a floating request card containing the victim's blood type and navigation link. Alerts for hazardous situations (`assault`, `fire`) are suppressed from bystander channels to ensure safety.

### 7. Companion First Responder App (`/responder`)
*   **Telemetry HUD**: Streams simulation speed (km/h), remaining distance, and ETA calculations in real-time as the vehicle approaches the victim.
*   **Live Triage Sync**: Synchronizes the avatar's triage chat log to the driver's console using WebSockets so responders are prepared upon arrival.

## 🛠️ Technology Stack

*   **Frontend**: React 19, Vite, Leaflet.js, Context API, Vanilla CSS (Glassmorphism design tokens)
*   **Backend**: Node.js, Express, Socket.io, Google Gen AI SDK, Helmet (Security), CORS
*   **Database**: MongoDB Server (Mongoose ODM, GeoJSON 2dsphere index)
*   **APIs**: Web Speech API, HTML5 Geolocation API, WebRTC MediaRecorder, Gemini Flash model

## ⚡ Setup & Launch Instructions

### Prerequisites
1. **Node.js**: Verify installation using `node --version` (v18+ recommended).
2. **MongoDB**: Ensure local database server is running.

### 1. Database Seeding
Connect to MongoDB and populate the 5 demo victim profiles (with Indian Names & preset locations):
```bash
# Navigate to project root
cd "C:\Users\nandh\.gemini\antigravity\scratch\smart-emergency-rescue"

# Start the database service
Start-Service MongoDB

# Seed the database
npm run seed
```

### 2. Configure Environment Variables
Open `server/.env` and specify credentials:
```
PORT=5000
MONGODB_URI=mongodb://localhost:27017/emergency_rescue
GEMINI_API_KEY=your_gemini_api_key_here
CLIENT_URL=http://localhost:5173
```
*Note: If GEMINI_API_KEY is left empty, the local rule-based fallback AI will handle classification automatically.*

### 3. Launch Development Servers
Launch both the backend API and frontend client concurrently:
```bash
npm run dev
```

*   👉 **Rescuer Dashboard**: [http://localhost:5173/dashboard](http://localhost:5173/dashboard)
*   👉 **First Responder HUD**: [http://localhost:5173/responder](http://localhost:5173/responder)
